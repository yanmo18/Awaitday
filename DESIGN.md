# Awaitday - 设计思考

## 产品灵魂

**核心意象**：一本精心设计的台历，每一页都是对未来的期待。

这个产品不是为了"提醒"，而是为了"期待"。用户打开它时，看到的不应该是冷冰冰的数字，而是一种仪式感——像翻开精心设计的日历，看着重要的日子一点点靠近。

## 气质与意象

**关键词**：仪式感、期待、温暖、精致

**场景描述**：
- 早晨醒来，打开App，看到生日倒计时的翻牌数字轻轻翻转，像老式翻页钟一样优雅
- 圆环进度缓缓填充，给人一种"时间在流动"的治愈感
- 不同主题带来不同氛围：生日是温暖的橙色，旅行是清新的蓝色，节日是喜庆的红色

## 视觉策略

**翻牌动画**：
- 核心视觉亮点，采用"翻页时钟"的隐喻
- 数字变化时，上半页向下翻转，露出新数字
- 翻转动画流畅优雅，duration 600ms，ease-out 缓动
- 微妙的阴影变化，营造立体感

**圆环进度**：
- 环形进度条，展示"已过去/总时长"的比例
- 使用 SVG 绘制，支持渐变色
- 圆环细线，精致不抢戏

**主题皮肤**：
- 每个主题不仅是配色，更是一种"氛围"
- 使用 Swiper 切换主题，每个主题卡片预览效果
- 主题影响：背景、圆环颜色、翻牌背景、强调色

## 字体排版

**数字字体**：系统等宽字体（monospace），确保翻牌动画数字对齐

**页面标题**：text-2xl font-semibold

**倒计时名称**：text-lg font-medium

**说明文字**：text-sm text-gray-500

## 动效与交互

**翻牌动画**：
- 触发时机：倒计时数字变化时（秒/分/时/天）
- 动画参数：transform: rotateX(-180deg), duration: 600ms, ease-out
- 视觉效果：上半页向下翻转，露出新数字
- 立体感：翻牌时添加 box-shadow 变化

**圆环进度**：
- 初始化时从 0% 动画到目标值
- 动画参数：duration 1000ms, ease-out

**主题切换**：
- Swiper 左右滑动切换主题
- 切换时背景色平滑过渡（transition: background-color 300ms）
- 切换后全局应用主题配色

## 页面结构

### 首页 - 倒计时列表
- 顶部：标题 "我的倒计时"
- 中间：倒计时卡片列表（每个卡片包含：翻牌动画 + 圆环进度 + 名称 + 日期）
- 底部：TabBar 导航

### 添加页 - 创建倒计时
- 表单：名称输入、日期选择、主题选择
- 提交按钮

### 主题页 - 皮肤切换
- Swiper 展示不同主题
- 每个主题卡片：主题名称 + 预览效果 + 应用按钮

## 导航结构

**TabBar 配置**：
1. 首页（倒计时列表）- icon: clock
2. 添加（创建倒计时）- icon: plus-circle
3. 主题（皮肤切换）- icon: palette

## 设计禁忌

- ❌ 禁止使用冷冰冰的纯数字展示
- ❌ 禁止翻牌动画生硬、卡顿
- ❌ 禁止主题配色冲突、刺眼
- ❌ 禁止圆环进度过粗、抢戏
- ❌ 禁止使用灰色为主的"工具感"配色
- ❌ 禁止扁平无趣的卡片设计（必须有质感）

## 技术实现要点

**翻牌动画**：
- 使用 CSS transform: rotateX 实现 3D 翻转
- 使用 perspective 创建 3D 空间
- 数字拆分为上下两半，分别翻转

**圆环进度**：
- SVG circle + stroke-dasharray + stroke-dashoffset
- 计算公式：offset = circumference * (1 - progress)

**主题切换**：
- Zustand 存储当前主题
- 主题切换时更新全局 CSS 变量
- 本地存储保存用户选择

**数据存储**：
- 使用 Taro.getStorage / setStorage 保存倒计时数据
- 数据结构：{ id, name, date, theme, createdAt }
